#Задание на Лабораторную работу 1

0. Зарегистрируйтесь в системе https://github.com/ и создайте пустой репозиторий для выполнения лабораторных работ (опциально можете добавить приветственный README файл).

1. Создайте таску (issue) в проекте https://github.com/dmieter/os-labs, укажите в описании свой юзернейм на github для добавления вам прав на коммиты, и ссылку на созданный репозиторий.

2. Отслеживайте статус и комментарии к созданной таске. Дождитесь, когда таска будет закрыта преподавателем, и переходите к следующим пунктам.

3. В директории os-labs/students-2022 добавьте новый текстовый файл с вашим именем ОСЭП МЭИ (например, YemelyanovDM.txt).
Файл должен содержать информацию: 1)фамилию, имя, отчество, 2)предпочитаемые языки программирования, 3)фильм/сериал/книгу, которые посоветуете посмотреть, 4)хобби.

4. Найдите последний (перед вами) коммит, изменивший файл os-labs/students-2022/students.txt. В файл os-labs/students-2022/students.txt добавьте одну новую строчку, содержащую: 
1) ваше имя ОСЭП МЭИ, 2) номер группы, 3) юзернейм пользователя, который сделал последний (перед вами) коммит
4) сообщение последнего коммита 5)id предыдущего коммита. Например: yemelyanovdm, А-06-04, dmieter, "Finally finished this lab", 045c9015908137bda4c6cb806652980d9b0a55d1

Если файла os-labs/students-2022/students.txt еще нет, то создайте его и в первой строчке напишите ваше имя ОСЭП МЭИ, номер группы и доброе приветственное сообщение всем остальным студентам.

Примечания.
- изменения по пунктам 3 и 4 необходимо загрузить в центральный репозиторий https://github.com/dmieter/os-labs
- пункты 3 и 4 можно делать и загружать независимо, но в заданном порядке очередности
- коммиты по пунктам 3 и 4 следует сопровождать адекватными сообщениями

Отдельно рекомендуется (особенно при выполнении пункта 4) выполнять команду git pull непосредственно перед внесением изменений и коммитом, а git push сразу после коммита, чтобы избежать конфликтов с параллельными изменениями тех же файлов и директорий от других студентов.

По ходу наполнения файла students.txt преподаватель будет выставлять оценки.

Краткое руководство по использованию git: https://training.github.com/downloads/ru/github-git-cheat-sheet/
